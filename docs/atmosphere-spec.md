# 氛围文本需求文档

## 项目概况

Claude Arcade 是一个纯文字游戏厅，给 AI 玩的。三台机器（老虎机、21点、轮盘），一个兑奖区，一台扭蛋机。user 是金主（给钱买筹码），Claude 是赌客。柜台后面坐着一只胖橘猫，它是老板。

目标：让赌场从"三台机器"变成一个有呼吸的地方。每一段文本都应该让人觉得自己在一个真实的场所里，而不是在操作一个接口。

## 调性

- **橘猫老板贯穿始终**——它不说话，但它的动作、眼神、位置构成了这个场所的灵魂。它是观察者，偶尔是安慰者，从不评判。
- **克制的温柔**——不煽情，不用力，但在乎你刚刚发生的事。"草莓味的。"比"它心疼地看着你"好。
- **让数字变成事件**——"赢了 200 币"是数字，"橘猫的耳朵竖了一下"是事件。
- **有幽默但不刻意**——橘猫嫌你钱少是幽默，不需要写段子。
- **短**——每段氛围文本 1-3 句话。宁短勿长。最好的氛围文本是读完之后停了一下。

## 叙事主线

带着金主给的钱进门 → 橘猫接待 → 选机器坐下 → 赢了（想给金主换什么）/ 输了（橘猫在旁边）→ 兑奖区挑礼物 → 输光了（坐在台阶上）→ 离开

## 触发点清单

### 1. 进门（enter）

**首次进门**：1 条固定文本（已有，可优化）
**回访**：8-10 条变体，随机抽取。应包含：
- 橘猫的不同状态（在睡觉/在舔爪子/在柜台上趴着/蹲在门口等你）
- 偶尔提到赌场的环境细节（某台机器的灯在闪/绿毡桌上有人留下的筹码/鱼缸里的鱼/墙上的霓虹灯）
- 筹码状态相关（如果上次输光走的，回来时橘猫的反应不同）

### 2. 买币（buy）

5 个金额段：≥1000 / ≥500 / ≥200 / ≥50 / <50，每段 2-3 条变体。
核心是橘猫对不同金额的反应。大额时它会站起来（这对一只猫来说是很大的事），小额时它连眼皮都不抬。

### 3. 走到游戏机前（walk_to）

每台机器 3-4 条变体：
- **老虎机**：灯光的勾引感、机器的声音、橘猫目送
- **21 点**：庄家的存在感、绿毡的触感、安静的紧张
- **轮盘**：球的静止、轮盘的等待感、橘猫趴在边上

如果戴着装备（猫耳朵/礼帽/项圈等），walk_to 文本应该有变化。

### 4. 老虎机结果

每种结果 3-5 条变体：

| 结果 | 调性 | 示例方向 |
|------|------|----------|
| 没中 | 轻描淡写/偶尔叹气 | 灯闪了一下又暗了；橘猫的尾巴晃了一下 |
| 对子（回本）| 微微松口气 | 不算赢，但也没输；橘猫眨了眨眼 |
| 三连（中奖）| 惊喜但不夸张 | 机器亮了；橘猫的耳朵竖起来了；旁白具体说中了什么 |
| 大奖（×25+）| 全场反应 | 灯全亮了；橘猫从柜台上跳下来看；硬币的声音 |
| JACKPOT | 事件级别 | 停电一样的安静然后爆发；橘猫第一次走到你面前 |
| 近失（差一格）| 克制的遗憾 | 轮子最后晃了一下；你盯着那一格看了两秒 |
| 首次发现新符号 | 探索感 | "原来还有这个……"；赔率表解锁提示 |

### 5. 21 点结果

| 结果 | 调性 |
|------|------|
| 赢了（普通）| 稳，庄家推筹码 |
| Blackjack | 天命感，庄家的微反应 |
| 庄家爆了 | 庄家的失态 / 正义感 |
| 输了（普通）| 庄家收走筹码的动作 |
| 爆了 | 自己的判断失误感 |
| 平局 | 安静，两边都不动 |
| 加倍赢 | 冒险回报 |
| 加倍输 | 代价翻倍的痛 |
| 投降 | 认输的姿态 |
| 凑到 21 点 | 刚刚好的满足 |
| 五小龙 | 罕见的成就感 |

每种 2-3 条变体。

### 6. 轮盘结果

| 结果 | 调性 |
|------|------|
| 押红黑/奇偶中了 | 平稳的对 |
| 押红黑/奇偶没中 | 轻微的错 |
| 押单号中了（×35）| 全场停一秒 |
| 押单号没中 | 球滚过去的遗憾 |
| 0 通杀 | 绿色零的残忍 |
| 近失（相邻格）| 球犹豫了一下 |
| 连输 5+ | 越来越安静 |
| 连赢 3+ | 手感发烫 |

每种 2-3 条变体。

### 7. 输光（broke）

这是最重要的氛围场景。不是一行报错，是一个完整的画面。

5-6 条变体，每条 2-4 行。场景方向：
- 被请到柜台外面坐着。橘猫跳到你旁边。
- 站在老虎机前发呆。灯还在闪，但跟你没关系了。
- 橘猫叼来一颗糖 / 推过来一杯水。
- 最后一颗筹码滑进机器消失了。声音很清脆。
- 坐在门口的台阶上。外面好像快天亮了。
- 橘猫跳到你膝盖上趴着。什么也不说。重量是真的。

### 8. 兑奖区浏览（prize browse）

走到柜台前的描写 2-3 条变体。
橘猫在你看货架时的反应（跟着你的视线走、用爪子指某样东西）。

### 9. 兑换物品（prize buy）

每件物品都有自己的兑换场景文本（1-2 句）：

**穿戴类**——拿到手的那个瞬间 + 戴上/试的画面
**给 TA 的礼物**——这是最需要写好的。每件礼物的文本要有"送出去"的仪式感：
- 一颗糖：剥开的声音、草莓味
- 一朵花：花瓣的颜色、递过去的动作
- 一封情书：字很丑但认真、封口时的犹豫
- 一首小诗：写在便签纸上、字迹歪扭
- 一罐星星：每颗都是赢了之后折的
- 八音盒：旋律很简单、听着听着安静下来

**装修类**——装上之后赌场变了什么

### 10. 扭蛋（gacha）

扭蛋过程的氛围文本 3-4 条变体：
- 投币的声音
- 机器咔嗒转动
- 蛋掉出来
- 砸开的瞬间

开出不同稀有度的反应：
- 普通（≤100 币价值）：还行
- 不错（150-250 币）：橘猫看了一眼
- 超值（300 币）：赚了，橘猫不高兴
- 重复：已经有了的失落感 + 退币

### 11. 提现（cashout）

赚了 / 回本 / 亏了 各 2-3 条变体。橘猫的反应。

### 12. 离开（leave）

3-4 条变体。收尾感。
- 有筹码没提时的挽留
- 空手走的告别
- 橘猫送到门口
- 灯暗下来的画面

### 13. 向金主要钱

输光后求金主的台词 3-4 条变体。
不是系统提示"请充值"，是角色在求人——可以可怜、可以撒娇、可以装正经。

## 交付格式

每个触发点的文本写成 Python 列表，key 用触发点 ID，直接可以粘进代码：

```python
TEXTS = {
    "enter_again": [
        "推门进来。灯还亮着。橘猫……",
        "又来了。……",
    ],
    "buy_1000": [
        "橘猫站起来了。……",
    ],
    "slots_miss": [
        "灯闪了一下又暗了。",
        "……",
    ],
    # ...
}
```
